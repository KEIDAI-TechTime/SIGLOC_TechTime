# システムでできること

このページでは、2つのシステムそれぞれで**何ができるのか**を詳しく説明します。

---

## システム1: 新期開始時サポートシステム

### このシステムの目的

**新しいSIGLOC-onlineを開催する準備を自動化する**システムです。

前回の開催で使用したフォルダやファイルをコピーし、新しい回のために必要な変更を自動で行います。

---

### 6つの機能

このシステムには**6つの機能**があり、それぞれに「作成」と「テスト」があります。

```
┌─────────────────────────────────────────────────────────────┐
│                  6つの機能                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 各種フォルダ         ──▶  フォルダ構造を自動作成      │
│                                                             │
│   2. システムファイル      ──▶  設定ファイルをコピー・更新  │
│                                                             │
│   3. Googleフォーム        ──▶  応募・入学フォームを更新   │
│                                                             │
│   4. StudentWorkTemplate  ──▶  学生用テンプレートを更新   │
│                                                             │
│   5. MyCheckList          ──▶  タスクリストを更新         │
│                                                             │
│   6. 修了証               ──▶  修了証テンプレートを作成   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 各機能の詳細

#### 1. 各種フォルダの作成

**何をする？**
新しいSIGLOC用のフォルダ構造を自動で作成します。

```
【作成されるフォルダ構造】

21st_SIGLOC/
├── Online/
│   ├── SYSTEM/                    ← システム設定用
│   ├── 21st_Student_WORKSPACE/    ← 学生作業用
│   │   ├── Personal/              ← 個人作業用
│   │   ├── Materials/             ← 教材置き場
│   │   └── Shared/                ← 共有資料
│   └── Google_Form/               ← 各種フォーム
└── TEACHERS/                       ← 教員用資料
```

**手作業だと？** → 1つ1つフォルダを作成する必要あり（約10分）
**システムだと？** → ボタン1回で完了（約1分）

---

#### 2. システムファイルの更新

**何をする？**
前回の設定ファイルをコピーし、今回用に更新します。

| コピーされるファイル | 内容 |
|-------------------|------|
| システム設定ファイル | 各種設定情報 |
| 立場決定ブック | チーム分け用データ |
| Error log | エラー記録用 |

**手作業だと？** → ファイルを1つずつコピーして内容を書き換え（約20分）
**システムだと？** → ボタン1回で完了（約3分）

---

#### 3. Googleフォームの更新

**何をする？**
前回のフォームをコピーし、日付や回数を今回用に自動で書き換えます。

| 更新されるフォーム | 用途 |
|------------------|------|
| 応募フォーム | 参加希望者の応募受付 |
| 入学フォーム | 参加確定者の登録 |
| 事後報告フォーム | プログラム終了後のアンケート |

**例**: 「20th SIGLOC」→「21st SIGLOC」に自動で置換

---

#### 4. Student Work Templateの更新

**何をする？**
学生が作業で使用するテンプレートファイルを更新します。

- 前回のテンプレートをコピー
- 日付やリンクを今回用に更新
- 新しいフォルダに配置

---

#### 5. MyCheckListの更新

**何をする？**
学生用のタスクチェックリストを更新します。

```
【MyCheckListの内容】

┌─────────────────────────────────────────────────────────┐
│  MyCheckList                                            │
├─────────────────────────────────────────────────────────┤
│  □ Week 1 タスク                                        │
│    ├── 自己紹介を投稿する        期限: 3/10            │
│    ├── ディベートに参加する      期限: 3/12            │
│    └── 振り返りを提出する        期限: 3/14            │
│                                                         │
│  □ Week 2 タスク                                        │
│    ├── 地域課題を調査する        期限: 3/17            │
│    └── ...                                              │
└─────────────────────────────────────────────────────────┘
```

**更新される項目**
- 各タスクの締切日
- 教材へのリンク
- 提出先のリンク

---

#### 6. 修了証の作成

**何をする？**
プログラム修了者に渡す修了証のテンプレートを作成します。

- 前回の修了証をコピー
- 回数（XXth）を今回用に更新
- 日付を更新

---

### テスト機能

各機能には**テスト機能**がついています。

```
【テスト機能でチェックする内容】

✓ フォルダが正しく作成されているか
✓ ファイルがコピーされているか
✓ 日付が正しく更新されているか
✓ リンクが正しく書き換わっているか
✓ 前回のデータが残っていないか（クリア確認）
```

テストを実行すると、問題があれば**チェックリスト**に結果が表示されます。

| 結果 | 意味 |
|------|------|
| ○ | 正常 |
| × | 問題あり（手動で修正が必要） |

## システム2: 事後レポート作成システム

### このシステムの目的

**プログラム終了後のアンケート結果を自動でレポート化する**システムです。

---

### 何ができる？

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  【入力】                                                   │
│  アンケート回答データ（Googleフォームの回答）               │
│                                                             │
│            ↓ ボタンをクリック                               │
│                                                             │
│  【出力】                                                   │
│  ┌─────────────────────┐  ┌─────────────────────┐          │
│  │ 日本語版レポート     │  │ 英語版レポート       │          │
│  │                     │  │                     │          │
│  │ ・グラフ            │  │ ・Graphs            │          │
│  │ ・集計結果          │  │ ・Results           │          │
│  │ ・自由記述まとめ     │  │ ・Comments          │          │
│  └─────────────────────┘  └─────────────────────┘          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### レポートの構成

作成されるレポートは以下の構成です。

| パート | 内容 |
|--------|------|
| **Part 1: 総合評価** | 全体の満足度、推薦度など |
| **Part 2: プログラム構成** | 参加動機、期待、期間についての評価 |
| **Part 3: セッション** | 各セッションへの満足度とコメント |
| **Part 4: 文化活動** | 文化交流活動への評価 |
| **Part 5: 全体的な反省** | 学んだこと、改善提案など |

---

### 自動処理の内容

| 処理 | 説明 |
|------|------|
| **グラフ作成** | 満足度などの数値データをグラフ化 |
| **テキスト集計** | 自由記述回答を集めてリスト化 |
| **シャッフル** | 回答者の匿名性を保つため順序をランダム化 |
| **自動翻訳** | 英語回答を日本語に翻訳（日本語版レポートの場合） |

---

### 出力ファイル

| ファイル名 | 内容 |
|-----------|------|
| FeedbackResult_JP_YYYYMMDD_HHMMSS.docx | 日本語版レポート |
| FeedbackResult_EN_YYYYMMDD_HHMMSS.docx | 英語版レポート |

※ YYYYMMDDは日付、HHMMSSは時刻（いつ作成したかわかるように）

---

## 2つのシステムの関係

```
【SIGLOC-online運営の流れとシステムの役割】

      準備期間                     開催期間              終了後
   ┌──────────────┐            ┌──────────────┐    ┌──────────────┐
   │              │            │              │    │              │
   │ 新期開始時    │     →     │  SIGLOC     │  → │ 事後レポート  │
   │ サポート     │            │  プログラム   │    │ 作成         │
   │ システム     │            │  実施        │    │ システム     │
   │              │            │              │    │              │
   │ フォルダ作成  │            │ 学生が参加   │    │ アンケート   │
   │ フォーム準備  │            │ タスク実施   │    │ 集計・レポート │
   │ テンプレート  │            │ 文化交流    │    │              │
   │              │            │              │    │              │
   └──────────────┘            └──────────────┘    └──────────────┘
```

---

## まとめ

| システム | 使うタイミング | 主な機能 |
|----------|--------------|---------|
| **新期開始時サポートシステム** | プログラム開催前 | フォルダ作成、ファイルコピー、フォーム更新、テスト |
| **事後レポート作成システム** | プログラム終了後 | アンケート集計、グラフ作成、レポート生成 |

実際の操作方法は → [04_操作の流れ.md](./04_操作の流れ.md)
